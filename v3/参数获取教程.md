## 第一步：获取 Cookie 🍪

### 操作步骤

1. **登录后**，在教务系统的**任意页面**（比如首页）
2. 按键盘 `F12` 打开开发者工具（或者右键 -> 检查）
3. 点击顶部的 **`Network`** (网络) 标签
4. **刷新页面** (按 `F5` 或点击刷新按钮)
5. 你会看到很多请求出现在左侧列表

### 找到 Cookie

6. 在左侧请求列表中，**随便点击一个请求**（建议点第一个）
7. 右侧会出现详细信息，找到 **`Headers`** (标头) 标签
8. 往下滚动，找到 **`Request Headers`** (请求标头) 区域
9. 找到 **`Cookie:`** 这一行

```
Cookie: JSESSIONID=xxx; fid=44165; 44165userinfo=xxx; ...
```

### 复制 Cookie

10. **鼠标选中** `Cookie:` 后面的**整行内容**（从第一个字符到最后一个分号）
11. **右键 -> 复制** (或按 `Ctrl+C`)

### 示例

```
JSESSIONID=0B49E2937F6991173131C1D30CB479D7; fid=44165; 44165userinfo=30cfb40d9fa2a06f6941b0244f6fccd7c49d67c0c30ca5043e655d84b33e56a2e74eecc06c08c425eb292109ae4dfee4c0433867ead52763518133e6115af878; 44165UID=404128970; _uid=404128970; 44165enc=6B5C79CE7694CD26D987874C3D30CE09; uf=d9387224d3a6095bd65b7cee38e5e815bf4b61bd53344b7c05a51c5f6a567993feeb727ecc9f90f60230ae2d711aadcb273683571faa0dcd88b83130e7eb4704e0cc755ad25f904e0246270a56330e6cbfecd4077347b641692c4351b857de90c8a31edf52a51c45; UID=404128970; vc=6B5C79CE7694CD26D987874C3D30CE09; vc2=93D5AB486DBA6896F6005DB66410AD5D; _d=1780287021208; vc3=euuh5ebp95e5y5l5ZM5nxY6Tpy6QBtLv2v%2BQdOX8oRIXB0dUvKKoe%2BhUDincbinssU1PC5A7yu%2BQ4qlBEYj2acbXPWFs8UbGzUpyqH1UiiGhSflZElKRij4BRf5r4fILiOhh5ZfaraEYezTrt25iauB%2BKqXyTMvNoDAEpTLRVz8%3D8a23386a4f7bea16d520d7b716e5596d; SERVERID=137
```

### ⚠️ 注意事项

- **复制整行**，不要只复制一部分
- 不要复制 `Cookie:` 这几个字，只要后面的值
- Cookie 很长(几百个字符)，这是正常的
- Cookie 会过期，一般几小时后需要重新获取

## 第二步：获取课程参数 📚

> 需要获取: `kcxx`、`jx0404id`、`kcid`

### 操作步骤

1. **保持 F12 开发者工具打开**
2. 在 **`Network`** 标签下，点击左上角的 **🗑️ 清除** 按钮（清空之前的请求记录）
3. 在教务系统页面，**进入选课页面**（点击"网上选课"或类似入口）
4. **找到你想抢的课程**
5. **点击该课程的"选课"按钮**（会弹出确认框，先不要点确定！）

### 找到关键请求

6. 在开发者工具的请求列表中，会出现很多新请求
7. **找到名为 `xxxkOper` 的请求**（有3个x）

```
名称              方法    类型
xxxkOper         GET     xhr
```

8. **点击这个 `xxxkOper` 请求**

### 方法一：从 URL 中提取（最简单）

9. 右侧会显示请求详情，找到 **`Headers`** 标签
10. 在最上方找到 **`Request URL`**（请求URL）

```
Request URL: https://inquiry.ecust.edu.cn/jsxsd/xsxkkc/xxxkOper?kcid=3E319A6B912E4B188E11F7899E2FC08D&cfbs=null&jx0404id=261117500010001
```

11. 从这个 URL 中提取参数：

| 参数 | 值 | 说明 |
|------|-----|------|
| `kcid` | `3E319A6B912E4B188E11F7899E2FC08D` | 32位大写字母+数字 |
| `jx0404id` | `261117500010001` | 通常15位数字 |

### 方法二：从 Payload 中查看（更清晰）

12. 在 `xxxkOper` 请求的详情中，找到 **`Payload`** (载荷) 标签
13. 或者找到 **`Query String Parameters`** (查询字符串参数) 区域

你会看到：
```
kcid: 3E319A6B912E4B188E11F7899E2FC08D
cfbs: null
jx0404id: 261117500010001
```

### 获取 kcxx (课程类别代码)

#### 方法A：从 Referer 中提取

14. 在同一个 `xxxkOper` 请求的 **`Headers`** 标签中
15. 找到 **`Referer`** 这一行

```
Referer: https://inquiry.ecust.edu.cn/jsxsd/xsxkkc/comeInXxxk?kcxx=117500010
```

16. `kcxx=` 后面的数字就是课程类别代码: **`117500010`**

#### 方法B：从 jx0404id 中提取

`jx0404id` 的前9位就是 `kcxx`:

```
jx0404id: 261117500010001
          └─前9位─┘
kcxx:     117500010
```

### 📝 记录参数

现在你应该得到了3个参数：

```json
{
    "kcxx": "117500010",
    "jx0404id": "261117500010001",
    "kcid": "3E319A6B912E4B188E11F7899E2FC08D"
}
```

## 第三步：填写配置文件 ✍️

打开 `config.json`，填入你刚才复制的内容：

```json
{
    "cookie": "这里粘贴你复制的完整Cookie",
    "base_url": "https://inquiry.ecust.edu.cn",
    "interval": 2,
    "max_retries": 3,
    "courses": [
        {
            "name": "Python程序设计",
            "kcxx": "117500010",
            "jx0404id": "261117500010001",
            "kcid": "3E319A6B912E4B188E11F7899E2FC08D",
            "enabled": true
        }
    ],
    "notification": {
        "sound": true,
        "duration": 1000
    }
}
```

### 字段说明

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| `name` | 否 | 课程名称，你自己起的，方便识别 |
| `kcxx` | 是 | 课程类别代码，9位数字 |
| `jx0404id` | 是 | 教学任务ID，15位左右数字 |
| `kcid` | 是 | 课程ID，32位大写字母数字组合 |
| `enabled` | 否 | 是否启用，默认 `true` |

## 🤔 常见问题

### Q1: 找不到 `xxxkOper` 请求？

**A:** 可能原因：
1. **没点"选课"按钮** - 必须点了才会发送请求
2. **点太快清空了** - 点按钮前先清空请求列表，点完后立即查看
3. **请求名称不同** - 有些学校可能叫 `xsxk` 或 `selectCourse`，找 `Method` 为 `GET` 且 URL 包含课程相关参数的

### Q2: Cookie 复制后脚本报错？

**A:** 检查：
1. 是否复制了**完整的 Cookie**（不要只复制一半）
2. 是否包含了 `Cookie:` 这几个字（不要包含，只要后面的值）
3. 是否有多余的空格或换行

### Q3: `kcid` 不是32位？

**A:** `kcid` 通常是32位的UUID格式（大写字母+数字），如果你看到的不是，可能：
1. 复制错了，重新检查
2. 你的学校系统格式不同（也可能是短一点的ID）

### Q4: 参数都对，但脚本一直显示"课程已满"？

**A:** 这是正常的！说明：
1. **参数配置正确**
2. **现在课程确实满了**
3. 脚本会持续监控，等有人退课就会自动抢

### Q5: 如何同时抢多门课？

**A:** 在 `courses` 数组里添加多个课程对象：

```json
"courses": [
    {
        "name": "Python程序设计",
        "kcxx": "117500010",
        "jx0404id": "261117500010001",
        "kcid": "3E319A6B912E4B188E11F7899E2FC08D",
        "enabled": true
    },
    {
        "name": "数据结构",
        "kcxx": "117500020",
        "jx0404id": "261117500020002",
        "kcid": "4F429B7CA13F5C299F22G8990F3GD19E",
        "enabled": true
    }
]
```

每门课重复"第二步"的操作，获取对应的参数。

### Q6: 还是不会怎么办？

**A:** 使用 v3.0 的**自动获取功能**：

```bash
python grab_v3.py --fetch
```

只需要提供 Cookie，脚本会自动拉取所有课程，你选择编号就行，无需手动抓取其他参数。

## 💡 小贴士

1. **Cookie要最新的** - 每次运行脚本前重新获取一次，确保新鲜
2. **参数是课程唯一的** - 每门课的 `kcid`、`jx0404id` 都不同
3. **先测试再挂机** - 跑一次看看参数对不对，显示"课程已满"就说明配置正确
4. **保护隐私** - Cookie 泄露等于账号泄露，不要分享给别人
