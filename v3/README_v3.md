# 华东理工大学自动选课脚本 v3.0 使用说明 📚

## 🎉 v3.0 新功能

### ✨ 自动获取课程信息
不再需要手动 F12 抓取 `kcid`、`jx0404id`！脚本可以自动从选课页面拉取所有课程，你只需要选择要抢的课程编号即可。

### 🔔 多种通知方式
- **提示音** - 抢到课后播放3次"哔哔"声
- **Webhook通知** - 支持钉钉机器人/企业微信机器人,人不在电脑前也能收到通知

### 🛡️ Cookie过期自动处理
- 运行中自动检测Cookie失效
- 失效后立即暂停并播放紧急提示音
- 引导你更新Cookie后按Enter继续,无需重启脚本

## 🚀 快速开始

### 方式一：自动获取课程(推荐)

1. **获取Cookie**
   ```bash
   # 浏览器登录教务系统 -> F12 -> Network -> 刷新 -> 复制Cookie
   ```

2. **更新配置文件** `config.json`
   ```json
   {
       "cookie": "这里粘贴你的Cookie",
       "base_url": "https://inquiry.ecust.edu.cn",
       "interval": 2
   }
   ```

3. **运行交互式课程选择**
   ```bash
   python grab_v3.py --fetch
   ```
   
   脚本会自动拉取所有可选课程,显示课程列表:
   ```
   找到 45 门课程:
   ------------------------------------------------------------
    1. Python程序设计              | 张老师     | 40/50
    2. 数据结构与算法              | 李老师     | 50/50 (已满)
    3. 计算机网络                  | 王老师     | 35/40
   ...
   ------------------------------------------------------------
   
   请输入要抢的课程编号(多个用逗号分隔,如: 1,3,5): 1,3
   ```

4. **开始抢课**
   ```bash
   python grab_v3.py
   ```

### 方式二：手动配置课程(兼容v2.0)

如果你已经有 `kcid`、`jx0404id` 等参数,直接在 `config.json` 里配置:

```json
{
    "cookie": "你的Cookie",
    "courses": [
        {
            "name": "Python程序设计",
            "kcxx": "117500010",
            "jx0404id": "261117500010001",
            "kcid": "3E319A6B912E4B188E11F7899E2FC08D",
            "enabled": true
        }
    ]
}
```

然后直接运行:
```bash
python grab_v3.py
```

## ⚙️ 配置详解

### 完整配置示例

```json
{
    "cookie": "你的完整Cookie字符串",
    "base_url": "https://inquiry.ecust.edu.cn",
    "interval": 2,
    "max_retries": 3,
    "courses": [
        {
            "name": "Python程序设计",
            "kcxx": "117500010",
            "jx0404id": "261117500010001",
            "kcid": "3E319A6B912E4B188E11F7899E2FC08D",
            "enabled": true
        }
    ],
    "notification": {
        "sound": true,
        "duration": 1000,
        "webhook": {
            "enabled": true,
            "url": "https://oapi.dingtalk.com/robot/send?access_token=xxx",
            "type": "dingtalk"
        }
    }
}
```

### 配置项说明

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `cookie` | 登录Cookie(必填) | - |
| `base_url` | 教务系统地址 | `https://inquiry.ecust.edu.cn` |
| `interval` | 检查间隔(秒) | `2` |
| `max_retries` | 单次请求重试次数 | `3` |
| `notification.sound` | 是否播放提示音 | `true` |
| `notification.duration` | 提示音时长(毫秒) | `1000` |
| `notification.webhook.enabled` | 是否启用Webhook | `false` |
| `notification.webhook.url` | Webhook地址 | - |
| `notification.webhook.type` | Webhook类型 | `dingtalk` |

## 🛡️ Cookie过期处理

### 运行中Cookie失效会怎样?

脚本检测到Cookie失效后会:
1. **立即暂停**抢课循环
2. **播放5次紧急提示音**(2000Hz高音)
3. **显示更新指引**:
   ```
   🚨 COOKIE 已过期！
   请按以下步骤操作:
   1. 打开浏览器重新登录教务系统
   2. 按 F12 -> Network -> 刷新页面 -> 复制 Cookie
   3. 更新 config.json 中的 cookie 字段
   4. 按 Enter 继续运行...
   ```
4. 等待你按 **Enter** 后自动重新加载配置,验证新Cookie,继续抢课

### 为什么要手动更新?

教务系统Cookie需要**真人登录**才能获取,脚本无法自动刷新。但v3.0的暂停+恢复机制让你不用重启脚本,已抢到的课程也不会丢失进度。

## 💡 使用技巧

### 1. 长时间挂机
- Cookie一般几小时后过期,使用v3.0的自动暂停功能,过期时更新即可继续
- 建议在选课前30分钟启动,确保Cookie新鲜

### 2. 多课程抢课策略
- 把最想要的课放在配置的前面(脚本按顺序检查)
- 不想抢的课设置 `"enabled": false` 即可临时禁用

### 3. 间隔时间设置
- **2秒**(推荐): 平衡速度和安全性
- **1秒**: 更快但可能被风控
- **5秒**: 更安全但可能错过退课窗口

### 4. Webhook通知场景
- 挂机时人不在电脑前,抢到课手机能收到通知
- 配合钉钉/企业微信,团队抢课可以互相提醒

---

## ⚠️ 注意事项

1. **Cookie时效** - v3.0会自动检测并引导更新,无需担心
2. **请求频率** - 不要把 `interval` 设置得太小(<1秒)
3. **选课时间** - 脚本只能在学校开放选课时工作
4. **网络稳定** - 确保网络连接稳定,避免频繁超时
5. **及时确认** - 抢到课后尽快登录教务系统确认

## 🐛 常见问题

### Q1: `--fetch` 模式获取不到课程?

**A:** 可能原因:
- Cookie失效 -> 重新获取
- 课程类别代码(kcxx)错误 -> 留空获取全部
- 选课页面结构变化 -> 反馈给我,我更新解析逻辑

### Q2: Webhook通知发送失败?

**A:** 检查:
- `url` 是否正确
- `type` 是否匹配(`dingtalk`/`wecom`)
- 机器人是否有权限(钉钉机器人需要加签/IP白名单等)

### Q3: Windows上提示音不响?

**A:** 确保:
- 系统音量未静音
- 运行环境是 Windows(Linux/Mac 需要修改提示音代码)

### Q4: 能在Linux/Mac上运行吗?

**A:** 可以,但需要注释掉 `winsound` 相关代码,或者安装跨平台的提示音库(如 `playsound`)
