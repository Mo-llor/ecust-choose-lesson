# 华东理工大学自动选课脚本使用说明 📚

## 🎯 功能特性

- ✅ **支持多门课程同时抢** - 一次配置，同时监控多门课
- ✅ **配置文件管理** - Cookie和课程信息独立配置，方便修改
- ✅ **智能错误处理** - 自动检测Cookie失效、网络常重试
- ✅ **日志记录** - 每次运行都会生成详细日志文件
- ✅ **成功通知** - 抢到课后自动播放提示音
- ✅ **智能重试** - 支持网络超时重试、动态调整请求间隔

## 📋 使用步骤

### 1. 获取Cookie

1. 打开浏览器，登录华东理工教务系统
2. 按 `F12` 打开开发者工具
3. 切换到 `Network (网络)` 标签
4. 刷新页面或随便点击一个链接
5. 在请求列表中点击任意一个请求
6. 在右侧 `Headers (标头)` 区域找到 `Request Headers (请求标头)`
7. 找到 `Cookie:` 这一行，**复制整行内容**（注意：是整行，包含很多 `key=value` 对）

### 2. 获取课程信息

#### 方法1：浏览器开发者工具抓包（推荐）

1. 在选课页面打开 `F12` 开发者工具
2. 切换到 `Network` 标签
3. 尝试手动选一次课（可以是任意课程）
4. 在 `Network` 中找到 `xsxkXxxk` 这个请求
5. 查看请求的 `Payload` 或 `Form Data`，记录以下参数：
   - `kcxx` - 课程代码（在URL参数中）
   - `jx0404id` - 教学任务ID
   - `kcid` - 课程ID

#### 方法2：从原脚本中查找

如果你之前的脚本能用，直接从里面复制这3个参数。

### 3. 编辑配置文件

编辑 `config.json`：

```json
{
    "cookie": "这里粘贴你的完整Cookie",
    "base_url": "https://inquiry.ecust.edu.cn",
    "interval": 2,
    "max_retries": 3,
    "courses": [
        {
            "name": "大学物理",
            "kcxx": "117500010",
            "jx0404id": "261117500010001",
            "kcid": "3E319A6B912E4B188E11F7899E2FC08D",
            "enabled": true
        },
        {
            "name": "高等数学",
            "kcxx": "117500020",
            "jx0404id": "261117500020002",
            "kcid": "4F429B7CA13F5C299F22G8990F3GD19E",
            "enabled": true
        }
    ],
    "notification": {
        "sound": true,
        "duration": 1000
    }
}
```

**配置说明：**

- `cookie`: 你的登录Cookie（必填）
- `interval`: 每次检查的间隔秒数（建议2-5秒，太快可能被封IP）
- `max_retries`: 每次请求失败后的重试次数
- `courses`: 课程列表
  - `name`: 课程名称（自己起的，方便识别）
  - `kcxx`: 课程代码
  - `jx0404id`: 教学任务ID
  - `kcid`: 课程ID
  - `enabled`: 是否启用（true/false）
- `notification.sound`: 是否播放成功提示音
- `notification.duration`: 提示音持续时间（毫秒）

### 4. 运行脚本

```bash
python grab_v2.py
```

## ⚠️ 注意事项

1. **Cookie会过期** - 如果脚本报告Cookie失效，需要重新获取Cookie并更新配置文件
2. **不要频繁请求** - `interval` 建议设置为2-5秒，太快可能被服务器封禁
3. **选课时间** - 请在学校规定的选课时间内运行脚本
4. **网络稳定** - 确保网络连接稳定
5. **及时确认** - 抢到课后请尽快登录教务系统确认

## 🐛 常见问题

### Q1: 提示"Cookie无效或已过期"

**A:** Cookie有时效性，一般几小时或一天后会失效。重新登录教务系统，按步骤1重新获取Cookie。

### Q2: 一直显示"课程已满"

**A:** 说明课程确实满了，脚本会持续监控。可以适当调整 `interval` 参数。

### Q3: 提示"响应不是JSON格式"

**A:** 可能是：
- Cookie已失效，返回了登录页面
- 服务器维护中
- 网络问题

查看日志文件中的详细响应内容来定位问题。

### Q4: 如何同时抢多门课？

**A:** 在 `config.json` 的 `courses` 数组中添加多个课程对象即可。脚本会依次尝试，成功的课程会自动移除，继续监控剩余课程。

### Q5: 如何临时禁用某门课？

**A:** 将该课程的 `enabled` 改为 `false` 即可，无需删除配置。
